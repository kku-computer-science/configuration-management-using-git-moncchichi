##หน้าที่ของแต่ละคน
### 1. ปภพ
- เขียนส่วนของอัลกอริทึม Bubble sort 
- ทำหน้าที่ตรวจสอบไฟล์และความถูกต้องก่อน merge  
- ดูแลปัญหา conflict ระหว่าง merge  

### 2. การติมา

- จัดการโครงสร้างโฟลเดอร์ เช่น AboutMyTeam, Project
- เขียนส่วนของอัลกอริทึม Quick sort

### 3. นิภาดา
- เขียนส่วนรับ input ข้อมูล
- ตรวจสอบ Pull Request ของเพื่อนร่วมทีม
- ทำหน้าที่ตรวจสอบไฟล์และความถูกต้องก่อน merge  

##วิธีทำงานร่วมกัน (workflow)
1. แต่ละคนสร้าง branch ของตัวเอง
2. ทำงานบน branch ของตัวเองเท่านั้น
3. เมื่อทำเสร็จ → commit → push ขึ้น GitHub
4. สร้าง Pull Request (PR) เพื่อรวมงานเข้ากับ main
4.1 ไปที่ GitHub
4.2 เลือก branch ของตัวเอง
4.3 คลิก Compare & Pull Request
4.4 รอเพื่อนกด Approve
5. เมื่อ PR ผ่าน → ทำการ merge เข้าสู่ main branch
6.หลัง merge → สมาชิกทุกคนต้อง pull main มาอัปเดตเครื่องตัวเอง

##วิธี merge
1. Checkout ไป branch main
2. Merge branch ของตัวเอง
3. ถ้าไม่มี conflict → merge สำเร็จ
4. Push ขึ้น GitHub

##วิธีแก้ conflict
1. เปิดไฟล์ที่มีปัญหา เช่น AboutMe.md
2. ลบส่วนที่ไม่ต้องการออก
3.เลือกข้อความที่ถูกต้อง
4. บันทึกไฟล์
5. ใช้คำสั่งนี้เพื่อจบการ conflict:
git add .
git commit -m "Fix merge conflict"
git push
